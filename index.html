The problem

Underwriting runs on what's reported. It should run on what's real.

Today

Cyber risk gets priced off self-attestations, point-in-time scans, and questionnaires that tell you a control is configured — not whether it's actually working. A checkbox and a functioning control look identical on paper. They aren't.

With Pulse

Pulse evaluates control integrity directly, not self-reported status — then standardizes the findings into signals that map to how underwriters already think about risk. Comparable across submissions. Current, not point-in-time.

How Pulse works

From raw evidence to underwriting intelligence.

  1. 01

    Collect evidence

    Pulse starts with identity security — beginning with Microsoft Entra ID — and is built as a modular platform that expands across the security stack over time.

  2. 02

    Evaluate control integrity

    Each control is assessed for whether it's actually functioning as intended, not just configured or reported as present.

  3. 03

    Generate standardized signals

    Findings are normalized into a consistent signal set — comparable across organizations, and current instead of point-in-time.

  4. 04

    Translate to underwriting intelligence

    Signals become meaningful assessments of risk, exposure, and insurance readiness — usable at submission, renewal, and across a portfolio.
